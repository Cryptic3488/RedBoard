The Problem

Film
Scattered Links
Hudl clips shared over group chat — no context, no notes, no targeting. Players scroll past them.
Stats
Raw Spreadsheets
CSV files emailed to players. No trend analysis, no personal framing, no context for what the numbers mean.
Wellness
Google Forms
Third-party form links texted daily. Coaches manually export and scan responses to gauge team readiness.
Playbook
Physical Binders
Printed plays that get lost, go out of date, and can't be updated in real-time before a game.
Communication
No Single Source
Content lives across Hudl, Google Drive, group chat, and email. Players don't know where to look.
The Result
Friction = Missed Reps
The harder it is for a player to access film or a play, the less likely they are to review it independently.
🏀

The Solution

A private, mobile-first web app that puts every coaching tool in one place — tailored to each player's role.

Technology Stack

Frontend
React 18 + Vite 5
TypeScript throughout. React Router v6 for client-side navigation. Tailwind CSS 3 for the design system.
Backend
Supabase
No custom server. Auth, database, file storage, and real-time subscriptions — all via Supabase's hosted platform.
Database
PostgreSQL
Row Level Security enforces access rules at the database level. No data can be read without an authorized JWT.
File Storage
Supabase Storage
Playbook PDFs, video clips, and player avatars stored in private buckets with policy-controlled access.
Stat Parsing
SheetJS (xlsx)
CSV and Excel files are parsed entirely client-side — no file ever touches a server. Column headers are auto-detected.
Design
Mobile-First
Designed for iPhone. Responsive to desktop. Light and dim modes. Denison brand red (#C8102E) throughout.

Authentication
Secure,
Role-Based
Access
Every user authenticates with Supabase Auth. On login, their role — coach or player — is loaded from the database and determines everything they can see and do.
  • Email + password authentication via Supabase
  • Two roles: Admin (coaches) and Player (roster)
  • Route guards redirect users to the correct view
  • JWT attached to every database request automatically
Player Home
Unified
Content Feed
The player's home screen surfaces everything in one place — a wellness nudge, a personal stat snapshot, and a live timeline of all coach-published content sorted newest-first.
  • Time-based greeting personalized to each player
  • Wellness nudge disappears once today's check-in is submitted
  • Stat snapshot shows hero stats with trend arrows vs. prior week
  • Feed merges film, stats, and playbook updates chronologically
Film Review
Hudl Clips
With Context
Coaches share Hudl links with written notes attached. Players see only the clips directed at them — a "just for you" badge marks individual posts. The red dot badge on the nav clears when they open the Film tab.
  • Team-wide or individual player targeting, enforced by RLS
  • Hudl iframe embeds play in-app — no redirect required
  • Real-time: new clips appear instantly via Supabase Realtime
  • Unread tracking per player via film_post_views table
Performance
Personal Stats,
In Context
Players see their own numbers — never a raw spreadsheet. Practice weeks are grouped and compared to the prior week. Games show deltas against career average. Coach annotations and personal goals are surfaced inline.
  • Game track: per-game with career average deltas
  • Practice track: weekly aggregates with trend arrows
  • Coach annotations appear as private notes per upload
  • Goal progress bars for admin-set stat targets
  • Hidden until coach clicks Publish — draft by default
Daily Check-In
How Is the
Team Feeling?
Coaches build custom check-in forms with rating (1–5), yes/no, and free-text questions. Players submit once per day. Coaches see team-level aggregates immediately — no export, no spreadsheet.
  • Form builder with drag-and-drop question ordering
  • Schedule forms on a calendar for future dates
  • Active form auto-rotates at midnight per schedule
  • One submission per player per day enforced in-app
Virtual Playbook
The Binder,
On Your Phone
Coaches upload plays as images or PDFs organized into folders. Players browse and open them in a full-screen lightbox. No printing, no lost binders — always up to date.
  • Folder-based organization (Offense, Defense, OOB, etc.)
  • Images and PDFs rendered inline — no app download required
  • Coaches control ordering and can update content instantly
  • Files stored in private Supabase Storage bucket

Security — Row Level Security

Table Player can read Admin can read
profiles Own row only All rows
film_posts Posts targeted to them All posts
stat_uploads Published only All uploads
stat_entries Own rows, published uploads only All entries
wellness_forms Active form only All forms
wellness_responses Own responses All responses
playbook_folders / files All (read-only) All (full CRUD)

Every table has RLS enabled with default-deny. The anon key is safe to expose — it cannot read anything without a valid, role-verified JWT. No custom server required.
